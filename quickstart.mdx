This guide walks you through a complete AdVerify setup from scratch. By the end you will have the Express server running, an API key created, and the Android SDK initialised in a real app.

<Note>
  You need a PostgreSQL database available before starting. Set the connection string as the `DATABASE_URL` environment variable on the server.
</Note>

<Steps>

<Step title="Deploy the server">

Clone the repository and install dependencies:

```bash
git clone https://github.com/yashmakhija/AdVerifySDK.git
cd AdVerifySDK/server
npm install
```

Create a `.env` file in `server/` with at minimum:

```bash .env
DATABASE_URL="postgresql://user:password@localhost:5432/adverify"
PORT=3000
FRONTEND_URL="http://localhost:3001"
ALLOWED_ORIGINS=""
```

Push the Prisma schema to your database and seed initial data:

```bash
npm run db:push
npm run db:seed
```

Start the development server:

```bash
npm run dev
```

Or build and run for production:

```bash
npm run build
npm start
```

Verify the server is running:

```bash
curl http://localhost:3000/
```

```json
{
  "status": "ok",
  "uptime": 1.23,
  "timestamp": "2026-01-01T00:00:00.000Z"
}
```

</Step>

<Step title="Create an API key">

Open the admin dashboard (default: `http://localhost:3001`) and navigate to **Keys**. Create a new API key for your Android app and note the generated key string â€” you will use it as `API_KEY` in the steps below.

Each API key is scoped to a single Android app (`packageName` + `appName`) and owns its own ads, PIN configuration, and analytics.

</Step>

<Step title="Add the SDK to your Android project">

Copy the `android-sdk/adverify` module directory into the root of your Android project, then declare it as a dependency.

In your root `settings.gradle`:

```groovy settings.gradle
include ':app', ':adverify'
```

In your app-level `build.gradle`:

```groovy build.gradle
dependencies {
    implementation project(':adverify')
}
```

The SDK requires `INTERNET` permission. Add it to your `AndroidManifest.xml` if it is not already present:

```xml AndroidManifest.xml
<uses-permission android:name="android.permission.INTERNET" />
```

</Step>

<Step title="Initialise the SDK">

Call `AdVerify.init()` once before any call to `show()`. The recommended place is your `Application` subclass:

```java MyApplication.java
import android.app.Application;
import com.adverify.sdk.AdVerify;

public class MyApplication extends Application {
    @Override
    public void onCreate() {
        super.onCreate();
        AdVerify.init(
            this,
            "YOUR_API_KEY",
            "https://your-server.com"
        );
    }
}
```

`init()` does two things:

- Validates that `apiKey` and `baseUrl` are non-empty (throws `IllegalArgumentException` otherwise)
- Reads the device's `ANDROID_ID` via `Settings.Secure` to use as the unique device identifier in all subsequent API calls

</Step>

<Step title="Show the ad flow">

Call `AdVerify.show()` from any `Activity` where you want the flow to appear â€” for example, your launcher activity's `onResume`:

```java MainActivity.java
import com.adverify.sdk.AdVerify;
import com.adverify.sdk.AdVerifyCallback;

public class MainActivity extends AppCompatActivity {

    @Override
    protected void onResume() {
        super.onResume();
        AdVerify.show(this, new AdVerifyCallback() {
            @Override
            public void onAdShown() {
                // Ad dialog is now visible
            }

            @Override
            public void onAdClosed() {
                // User dismissed the ad
            }

            @Override
            public void onAdClicked(String redirectUrl) {
                // User tapped the ad â€” redirectUrl is the destination
            }

            @Override
            public void onError(String message) {
                // Handle initialisation or network errors
            }
        });
    }
}
```

If you do not need lifecycle callbacks, use the shorter overload:

```java
AdVerify.show(this);
```

</Step>

<Step title="MT Manager: single-call smali injection">

If you are injecting into an existing APK using MT Manager rather than building from source, use `AdVerify.start(Activity)` instead. It reads the API key and server URL from `AndroidManifest.xml` meta-data, so no `Application` class changes are needed.

Add meta-data to your `AndroidManifest.xml` inside the `<application>` tag:

```xml AndroidManifest.xml
<application ...>

    <meta-data
        android:name="adverify.api_key"
        android:value="YOUR_API_KEY" />

    <meta-data
        android:name="adverify.base_url"
        android:value="https://your-server.com" />

</application>
```

Then inject this single smali line into the target activity's `onResume` or `onCreate`:

```smali
invoke-static {p0}, Lcom/adverify/sdk/AdVerify;->start(Landroid/app/Activity;)V
```

`start(Activity)` initialises the SDK from meta-data (if not already initialised) and immediately calls `show()`. It also sets `FLAG_SECURE` on the window to block screenshots and screen recording.

</Step>

<Step title="Verify end-to-end">

With the server running and the app on a device or emulator:

1. Launch the app â€” the SDK posts to `POST /api/sdk/init` with your device's `ANDROID_ID`.
2. If **PIN is enabled** in the admin panel for this API key, the PIN verification dialog appears. Tap **Generate PIN & Verify** to open the shortener link in the browser, complete the flow, and enter the received PIN.
3. Once verified (or if PIN is disabled), the SDK fetches ads from `GET /api/sdk/ads` and displays the ad dialog.
4. Open the admin dashboard and check **Dashboard** â€” you should see an impression recorded for your device.

<Tip>
  Use the admin panel's **User PINs** section to verify that your device ID appears with a `usedAt` timestamp after completing PIN verification.
</Tip>

</Step>

</Steps>
