## What is AdVerify?

AdVerify is a full-stack ad monetization system for Android apps. It gives you three things in one package:

1. **PIN verification gate** — before a user can use your app, they must complete a monetized verification flow (viewing ads or articles) to receive a unique PIN.
2. **Interstitial ad serving** — a programmatically rendered dialog shows ads fetched from your server, with impression and click tracking.
3. **Admin dashboard** — a Next.js web panel lets you manage API keys, ad creatives, PIN configuration, and view verified devices.

The Android SDK has **zero external dependencies** and renders all UI using Android's native `Canvas`, `Paint`, and `GradientDrawable` APIs, keeping your app lightweight.

## How it works

```
App Launch
    │
    ▼
SDK calls POST /api/sdk/init with deviceId + x-api-key
    │
    ├── pinEnabled=false OR pinVerified=true
    │       → Fetch ads → Show AdDialog
    │
    └── pinEnabled=true AND pinVerified=false
            → Show PinVerifyDialog
                │
                ├── User taps "Generate PIN & Verify"
                │       → Server creates shortener link
                │       → Browser opens (user earns PIN)
                │       → User enters PIN
                │       → POST /api/sdk/verify-pin
                │               └── Valid → dismiss → show ads
                │
                └── User taps "Exit" → app closes
```

## Get started

<Steps>
  <Step title="Deploy the server">
    Clone the repo, set your `DATABASE_URL`, and run `bun dev` to start the Express API.
    See [Server Setup](/server/setup).
  </Step>
  <Step title="Create an API key">
    Open the admin dashboard and create an API key for your app.
    See [API Keys](/dashboard/api-keys).
  </Step>
  <Step title="Add the SDK to your Android project">
    Copy the `adverify` module into your project and add it as a dependency.
    See [SDK Installation](/sdk/installation).
  </Step>
  <Step title="Initialize and show">
    Call `AdVerify.init()` in your `Application.onCreate()` and `AdVerify.show()` where needed.
    See [Initialization](/sdk/initialization).
  </Step>
</Steps>
